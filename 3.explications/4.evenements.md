# Gestion des événements

- Les événements en React sont gérés de manière similaire aux événements DOM, mais avec quelques différences notables.

### Syntaxe des événements

- En React, les noms des événements utilisent la camelCase au lieu de la minuscule. Par exemple, on utilise `onClick` au lieu de `onclick`.

```jsx
function MyButton() {
  return <button onClick={() => alert('Button clicked!')}>Click me</button>;
}
export default MyButton;
```

- On pourra passer une fonction comme gestionnaire d'événement. Il est important de noter que l'on ne met pas de parenthèses après le nom de la fonction, car on veut passer la référence de la fonction, pas le résultat de son appel.

```jsx
function handleClick() {
  alert('Button clicked!');
}
function MyButton() {
  return <button onClick={handleClick}>Click me</button>;
}
export default MyButton;
```

### Gestion des événements avec des paramètres

- Si vous avez besoin de passer des paramètres à votre gestionnaire d'événements, vous pouvez utiliser une fonction fléchée ou une fonction anonyme.

```jsx
function MyButton() {
  return <button onClick={(e) => handleClick(e, 'Hello World')}>Click me</button>;
}
function handleClick(event, message) {
  alert(message);
}
export default MyButton;
```

### Liste des événements courants

- Voici une liste de quelques événements courants en React :

  - `onClick` : déclenché lorsqu'un élément est cliqué.
  - `onChange` : déclenché lorsqu'une valeur d'entrée change.
  - `onSubmit` : déclenché lorsqu'un formulaire est soumis.
  - `onMouseEnter` : déclenché lorsque la souris entre dans un élément.
  - `onMouseLeave` : déclenché lorsque la souris quitte un élément.
  - `onKeyDown` : déclenché lorsqu'une touche du clavier est enfoncée.
  - `onKeyUp` : déclenché lorsqu'une touche du clavier est relâchée.
  - etc...
