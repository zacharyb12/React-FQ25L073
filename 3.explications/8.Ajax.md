# Ajax avec React

### Requetes HTTP
- il existe deux principales façons d'effectuer des requêtes HTTP dans une application React : l'API Fetch intégrée et une bibliothèque comme Axios.

### API Fetch
- L'API Fetch est une interface native pour effectuer des requêtes HTTP. Elle est basée sur des Promises et est donc asynchrone par nature.

```jsx
useEffect(() => {
  const fetchData = async () => {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  };

  fetchData();
}, []);
```

### Axios
- Axios est une bibliothèque tierce qui simplifie les requêtes HTTP. Elle offre une syntaxe plus concise et des fonctionnalités supplémentaires, comme l'annulation des requêtes.

```jsx
import axios from 'axios';

useEffect(() => {
  const fetchData = async () => {
    const response = await axios.get('https://api.example.com/data');
    console.log(response.data);
  };

  fetchData();
}, []);
```

### Pourquoi effectuer les requetes dans useEffect ?
- Les requêtes HTTP sont souvent effectuées dans le hook `useEffect` pour s'assurer qu'elles ne sont exécutées qu'après le rendu initial du composant.

- Cela évite les appels redondants à chaque rendu et permet de gérer les effets de bord de manière appropriée.

- Et permet de contrôler quand les données doivent être rechargées en fonction des dépendances spécifiées dans le tableau de dépendances de `useEffect`.

- Et de gérer le nettoyage des requêtes si le composant est démonté avant la fin de la requête.
